# Аналіз мультиязичності (i18n)

Проект використовує бібліотеку `next-intl` для забезпечення підтримки декількох мов.

## Конфігурація

1.  **Підтримувані мови**:
    *   `uk` (Українська) - мова за замовчуванням.
    *   `de` (Німецька).
    *   `en` (Англійська).

2.  **Файли конфігурації**:
    *   `src/i18n.ts`: Основна конфігурація. Визначає список локалей, дефолтну локаль та логіку завантаження повідомлень (messages) з JSON файлів.
    *   `src/middleware.ts`: Middleware для Next.js. Перехоплює запити та перенаправляє користувачів на відповідну локаль (наприклад, `/` -> `/uk`). Використовує стратегію `localePrefix: 'always'`, що означає, що префікс мови завжди присутній в URL.

## Файли перекладів

Переклади зберігаються у папці `messages/` у форматі JSON:
*   `messages/uk.json`
*   `messages/de.json`
*   `messages/en.json`

Структура JSON файлів ієрархічна, наприклад:
```json
{
  "home": {
    "hero": {
      "title": "...",
      "description": "..."
    }
  }
}
```

## Використання в коді

У компонентах (наприклад, `src/app/[locale]/page.tsx`) використовуються хуки від `next-intl`:
*   `useTranslations('namespace')`: Отримує функцію перекладу для певного простору імен (наприклад, 'home').
*   `useLocale()`: Отримує поточну активну локаль.

Приклад:
```tsx
const t = useTranslations('home');
<h1>{t('hero.title')}</h1>
```

## Маршрутизація

Всі сторінки знаходяться в динамічній папці `src/app/[locale]`, що дозволяє автоматично передавати параметр `locale` у компоненти сторінок.
